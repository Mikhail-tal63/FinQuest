هذه أهم 10 سيناريوهات تطبقوها في مشروع FinLife للـ MVP:

1. رسالة تحديث بيانات البطاقة

المصدر: Inbox
الدور: All
النوع: Fraud / Security

السيناريو:
وصلك إيميل يقول إن بطاقتك سيتم إيقافها خلال دقائق، ويطلب منك تحديث بيانات البطاقة عبر رابط.

الهدف:
تعليم المستخدم عدم إدخال بيانات البطاقة في روابط مشبوهة.

2. منحة دراسية وهمية

المصدر: Inbox
الدور: Student
النوع: Fraud

السيناريو:
وصلك إيميل يقول إنك حصلت على منحة دراسية، لكنه يطلب بيانات بطاقتك لاستلام المبلغ.

الهدف:
تعليم الطالب التحقق من الجهات الرسمية قبل مشاركة أي بيانات مالية.

3. تحديث بيانات الراتب

المصدر: Inbox
الدور: Employee
النوع: Fraud / Work Email

السيناريو:
وصلك بريد يدّعي أنه من قسم الموارد البشرية ويطلب تحديث بيانات الحساب البنكي لاستلام الراتب.

الهدف:
تعليم الموظف التحقق من رسائل العمل قبل إدخال معلومات مالية.

4. عميل يطلب رسوم تفعيل

المصدر: Inbox
الدور: Freelancer
النوع: Fraud / Freelance

السيناريو:
عميل يقول إنه سيدفع لك مبلغًا كبيرًا، لكنه يطلب منك دفع رسوم تفعيل أولًا لاستلام الدفعة.

الهدف:
تعليم صاحب العمل الحر أن دفع رسوم لاستلام المال علامة احتيال.

5. تقسيم الراتب أو المصروف

المصدر: Wallet
الدور: All
النوع: Budgeting

السيناريو:
وصل لك دخل جديد، وتحتاج أن تقرر كيف توزعه بين الفواتير، المصاريف، والادخار.

الهدف:
تعليم المستخدم التخطيط المالي وتوزيع الدخل بشكل صحيح.

6. فاتورة إنترنت مستحقة

المصدر: Bills
الدور: All
النوع: Bills

السيناريو:
لديك فاتورة إنترنت مستحقة اليوم، وتحتاج أن تقرر هل تدفعها الآن أو تؤجلها.

الهدف:
تعليم المستخدم ترتيب الأولويات ودفع الالتزامات المهمة.

7. طلب تحويل مالي عاجل

المصدر: Phone / Notifications
الدور: All
النوع: Transfers / Security

السيناريو:
وصلك إشعار أو رسالة من شخص يطلب تحويل مبلغ بسرعة ويقول إن الأمر عاجل.

الهدف:
تعليم المستخدم التحقق من هوية الشخص والتفاصيل قبل التحويل.

8. عرض شراء مغرٍ

المصدر: Phone / Notifications
الدور: All
النوع: Shopping / Budgeting

السيناريو:
ظهر لك عرض شراء جهاز بسعر منخفض جدًا ولفترة محدودة.

الهدف:
تعليم المستخدم مراجعة الميزانية والتمييز بين الحاجة والرغبة قبل الشراء.

9. هدف ادخار

المصدر: Wallet
الدور: All
النوع: Savings

السيناريو:
لديك هدف مالي مثل شراء لابتوب أو دفع رسوم أو تطوير عملك، وتحتاج أن تقرر كيف تدخر له.

الهدف:
تعليم المستخدم تخصيص مبلغ ثابت للادخار بدل انتظار ما يتبقى.

10. عرض استثمار سريع الربح

المصدر: Inbox
الدور: All
النوع: Fraud / Investment

السيناريو:
وصلك إعلان يقول: “ضاعف مالك خلال 48 ساعة فقط”.

الهدف:
تعليم المستخدم الحذر من الوعود المالية غير الواقعية وعروض الربح السريع.

Inbox:

5. عرض استثمار سريع الربح

Wallet:
6. تقسيم الراتب أو المصروف
7. هدف ادخار

Bills:

الاحتيال، الميزانية، الفواتير، التحويلات، الادخار، والقرارات اليومية.
